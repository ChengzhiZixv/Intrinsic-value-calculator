# 全局变量清单

本文档列出所有全局变量（常量），它们的作用、默认值、位置，以及如何调整。

---

## 📍 数据层（Layer 1）

### `THRESHOLD_MONTHS`
- **位置**：`src/valmod/data_layer/quality.py` 第 18 行
- **类型**：`int`
- **默认值**：`6`
- **作用**：财报超过此月数未更新则触发 Warning
- **如何调整**：直接修改文件中的 `THRESHOLD_MONTHS = 6`，改为你想要的月数

---

## 📍 假设引擎（Layer 4）

### `CAGR_CAP`
- **位置**：`src/valmod/assumptions/engine.py` 第 20 行
- **类型**：`float`
- **默认值**：`0.15`（15%）
- **作用**：历史 CAGR 上限，防止极端高增长
- **如何调整**：直接修改文件中的 `CAGR_CAP = 0.15`

### `CAGR_FLOOR`
- **位置**：`src/valmod/assumptions/engine.py` 第 22 行
- **类型**：`float`
- **默认值**：`-0.10`（-10%）
- **作用**：历史 CAGR 下限，负增长公司特殊处理
- **如何调整**：直接修改文件中的 `CAGR_FLOOR = -0.10`

### `DEFAULT_GROWTH`
- **位置**：`src/valmod/assumptions/engine.py` 第 24 行
- **类型**：`float`
- **默认值**：`0.05`（5%）
- **作用**：算不出 CAGR 时的默认增长率
- **如何调整**：直接修改文件中的 `DEFAULT_GROWTH = 0.05`，或通过 `config/analyst_overrides.yaml` 的 `explicit_growth_rate` 覆盖

## 📍 主流水线

### `MAX_RETRIES`
- **位置**：`src/valmod/pipeline.py` 第 41 行
- **类型**：`int`
- **默认值**：`3`
- **作用**：yfinance API 失败时的最大重试次数
- **如何调整**：直接修改文件中的 `MAX_RETRIES = 3`

### `RETRY_DELAY`
- **位置**：`src/valmod/pipeline.py` 第 42 行
- **类型**：`float`
- **默认值**：`2.0`（秒）
- **作用**：重试延迟的基础时间（指数退避：第 n 次重试延迟 = RETRY_DELAY * 2^n）
- **如何调整**：直接修改文件中的 `RETRY_DELAY = 2.0`

## 💡 调整建议

1. **优先通过 config 调整**：`DEFAULT_GROWTH`、`DEFAULT_PE_ANCHOR`、`DEFAULT_EV_EBITDA_ANCHOR`、`DEFAULT_EV_SALES_ANCHOR` 可通过 `config/analyst_overrides.yaml` 覆盖，无需改代码。

2. **告警阈值**：`TERMINAL_PCT_WARN`、`DIVERGENCE_THRESHOLD`、`DA_REVENUE_THRESHOLD`、`FCF_NI_RATIO_FLOOR` 等告警阈值建议根据实际使用经验调整。

3. **情景扰动幅度**：`GROWTH_SHOCK`、`RATE_SHOCK`、`G_SHOCK` 可根据市场波动性调整。

4. **重试配置**：`MAX_RETRIES`、`RETRY_DELAY` 通常不需要调整，除非 yfinance API 频繁失败。
