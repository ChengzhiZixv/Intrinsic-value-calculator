# 📁 项目结构完全说明（已实现版）

## 🎯 整体比喻
把这个项目想象成**一个工厂流水线**：
- 你输入一个股票代码（如 ORCL、AAPL、MSFT）
- 经过 8 道工序（8 层）
- 最后输出一份估值报告

**当前状态：所有 8 层已完整实现，可直接使用。**

**类比**：工厂的总控制台，按顺序调用每一道工序  
**工作流程**：
```
输入任意 Ticker（如 ORCL、AAPL）
→ Layer 1 拉数据
→ Layer 2 标准化
→ Layer 3 分类
→ Layer 4 生成假设
→ Layer 5 算估值
→ Layer 6 算情景
→ Layer 7 融合结果
→ Layer 8 生成告警
→ 输出最终报告
```

**你现在要做什么**：
- 已完整实现，可直接调用 `run_valuation(ticker)`
- 支持从 `config/analyst_overrides.yaml` 加载参数
- 支持重试机制（yfinance API 失败时）

## 📂 `src/valmod/models/` 文件夹（第 5 层：估值模型层）✅ **已实现**

**作用**：实际计算估值（这是核心！）

### `models/__init__.py`
**作用**：让 Python 知道这是一个子包  
**你现在要做什么**：已写好，不用管

### `models/dcf.py` ⭐⭐ ✅ **已实现**
**作用**：**DCF 估值模型**（自由现金流折现）  
**类比**：用"未来能赚多少钱"来算公司值多少钱  
**已实现功能**：
- 预测未来 5 年的自由现金流（FCF）
- 计算终值（5 年后的价值，Gordon Growth）
- 用折现率把所有未来现金流折现到现在
- 除以股本，得到每股估值
- 计算终值占比（如果 >70% → 警告）
- 输出：每股估值、终值占比、警告列表

**你现在要做什么**：已实现，可直接使用

### `models/reverse_dcf.py` ⭐ ✅ **已实现**
**作用**：**反向 DCF**（学习用）  
**类比**：反过来问"如果要撑起当前股价，未来需要增长多少？"  
**已实现功能**：
- 给定当前股价，反推需要多高的 FCF 增长率
- 反推第 5 年的 FCF 规模
- 对比隐含的 FCF margin 和历史 margin
- 输出：隐含增长率、隐含 FCF5、隐含 margin 对比

**你现在要做什么**：已实现，可直接使用

### `models/multiples.py` ⭐ ✅ **已实现**
**作用**：**相对估值模型**（PE、EV/EBITDA、EV/Sales）  
**类比**：用"同行公司值多少钱"来对比  
**已实现功能**：
- 计算 PE 估值（**使用稀释 EPS，与 MOOMOO 一致**）
- 计算 EV/EBITDA 估值（如果有 EBITDA 数据）
- 计算 EV/Sales 估值（如果有收入数据）
- 输出：各种相对估值结果

**你现在要做什么**：已实现，可直接使用

### `models/registry.py` ⭐ ✅ **已实现**
**作用**：**模型注册表**，统一调用所有模型  
**类比**：总调度，根据"可用模型列表"调用对应的模型  
**已实现功能**：
- 根据 `EnabledModels` 调用对应的模型（dcf、multiples、reverse_dcf）
- 汇总所有模型的结果
- 处理模型失败的情况（优雅降级）
- 输出：`ModelOutputs` 对象（包含所有模型的结果）

**你现在要做什么**：已实现，可直接使用

**已实现功能**：
